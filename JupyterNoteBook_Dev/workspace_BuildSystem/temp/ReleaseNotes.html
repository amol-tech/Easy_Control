For GT 155136
-------------
The modified mapping for BBCMPSEC will ensure that any change in the BBCMPSEC value on the same market and same ID_BB_UNIQUE, the existing ISID instance in the database is updated.
 
If the client implementation requires the history of such a change to be maintained, the TimeSeries will have to be configured for the same. The TS configuration has to be added to Conditional Timeseries xml (TSCONDNL) for the ISID instance where ID_CTXT_TYP = 'BBCMPSEC'. This will apply the TS option to BBCMPSEC.
  
Below is a sample configurtion that is to be added to the Conditional Timeseries xml (TSCONDNL) in FT_T_XCFG. Do note that if the client wishes to configure TimeSeries for BBCMPSEC, then all the messageTypes have to be mentioned in the configuration, as comma-separated values.
 
<occurrence table="ft_t_isid">
            <option id="ISID_1" messageTypes="BBGlobalEquity">
                        <attribute>ISS_ID</attribute>
                                    <where>
                                                            <field optr="EQUAL" multiValue="false" value="BBCMPSEC" name="FT_T_ISID.ID_CTXT_TYP"/>
                                    </where>
            </option>
</occurrence>
